Assuming you're actively employed and can spend 1–2 hrs/weekday + 4–5 hrs/weekend. Adjust pacing if you have more time.
WEEK 1 Core Java Depth + Multithreading Foundation ▼
  • Revise JVM internals — heap, stack, GC (G1, ZGC). Read Baeldung's GC series.
  • Collections deep dive — HashMap internals (hash collision, resize), LinkedHashMap, TreeMap, ConcurrentHashMap.
  • Java 8–17 features — Streams (collectors, flatMap, groupingBy), Optional, Records, Sealed classes.
  • Multithreading: Thread lifecycle, synchronized, volatile, happens-before. Practice writing a thread-safe singleton.
  • ExecutorService, CompletableFuture — write a small async pipeline. Practice: chaining thenApply, thenCompose, exceptionally.
  • Concurrency utilities — CountDownLatch, Semaphore, BlockingQueue. Write code examples for each.
  • LeetCode: 2–3 easy/medium HashMap/Array problems daily (warm-up habit).
WEEK 2 Spring Boot Deep Dive + Spring Security Core Stack ▼
  • Spring IoC — write a custom BeanFactoryPostProcessor. Understand bean lifecycle hooks.
  • AOP — implement a logging aspect using @Around. Understand JDK proxy vs CGLIB difference.
  • @Transactional — test all propagation types (REQUIRED, REQUIRES_NEW, NESTED). Understand self-invocation pitfall.
  • Spring Boot autoconfiguration — read source of one starter (e.g. DataSourceAutoConfiguration). Understand @Conditional.
  • Spring Security: Build a small app with JWT auth. Implement UserDetailsService. Add role-based @PreAuthorize.
  • OAuth 2.0 flows — draw out the Authorization Code flow on paper. Understand refresh token rotation.
  • LeetCode: Two Pointer + Sliding Window problems (5–6 total this week).
WEEK 3 JPA / Hibernate + SQL Mastery Data Layer ▼
  • JPA entity lifecycle states — transient, managed, detached, removed. Write code that demonstrates each.
  • N+1 problem — reproduce it, then fix it with JOIN FETCH and @EntityGraph.
  • Transaction isolation levels — READ_UNCOMMITTED to SERIALIZABLE. Understand dirty read, non-repeatable read, phantom read.
  • Optimistic locking — implement @Version field. Simulate a conflict and handle ObjectOptimisticLockingFailureException.
  • SQL: Window functions (ROW_NUMBER, RANK, LAG, LEAD), CTEs, recursive queries. Practice on LeetCode SQL section.
  • Database indexing — understand when composite index column order matters. Practice EXPLAIN ANALYZE on slow queries.
  • LeetCode SQL 50 — complete 10 medium SQL problems this week.
WEEK 4 Microservices Patterns + Kafka Expert Architecture ▼
  • Microservices patterns — read microservices.io. Be able to explain Saga (choreography vs orchestration) with a real example from your HCSC payment project.
  • Circuit Breaker — implement Resilience4j in a Spring Boot service. Configure fallback and half-open state.
  • API Gateway patterns — rate limiting, auth offloading, request aggregation.
  • Kafka deep dive — producer acks (0/1/all), consumer group rebalancing, offset management. Reproduce a rebalance scenario.
  • Spring Kafka — implement DLT (Dead Letter Topic) error handling. Write a retry consumer with @RetryableTopic.
  • Distributed tracing — revise Cloud Sleuth (you've used it). Understand trace ID propagation across services.
  • LeetCode: Binary Search problems (5–6). Start Trees (BFS/DFS).
WEEK 5 System Design (Intensive) Senior Rounds ▼
  • Read ByteByteGo chapters: URL shortener, rate limiter, notification system, consistent hashing.
  • Design a payment processing system — map your HCSC experience to the concepts (Fiserv integration, error handling, retries).
  • Design an e-filing portal (your Infosys work) — discuss scale, PDF generation service, async processing.
  • Caching deep dive — cache aside vs write-through. When to use Redis for session vs data caching. Cache invalidation strategies.
  • Database sharding vs partitioning — know the difference. Understand horizontal scaling limitations of RDBMS.
  • CAP theorem + PACELC — pick real examples (Kafka = AP, RDBMS = CP in typical config).
  • Practice talking through a design out loud for 30–40 mins (key skill for system design rounds).
WEEK 6 Design Patterns + Performance + Testing Depth ▼
  • SOLID — prepare a concrete code example for each principle (violation → fix). This comes up constantly.
  • Design patterns — implement Strategy (your ClearCarbon status cycle is a good fit), Decorator, Chain of Responsibility, Observer in Java. Don't just memorise — code them.
  • JVM performance — take a thread dump, analyse it. Simulate a deadlock and read the dump. Understand GC log format.
  • HikariCP pool tuning — understand maximumPoolSize, connectionTimeout, leakDetectionThreshold. When does pool exhaustion occur?
  • Testing mastery — write a test with TestContainers for a JPA repo. Write a MockMvc test for a secured endpoint. Practice ArgumentCaptor.
  • Revisit Elasticsearch — DSL queries (bool/filter/must), aggregations, Spring Data ES. Practise your PDF→JSON→ES story.
  • LeetCode: Tree + Graph problems (7–8). Attempt 2–3 Mediums on DP.
WEEK 7 Mock Interviews + Gap Filling Simulation ▼
  • Mock system design interview: design a "Control Report generation system" (maps directly to your Citi experience). Time yourself — 45 mins.
  • Mock coding: 2 LeetCode mediums under timed conditions (45 mins each).
  • Mock behavioural: Answer all 10 behavioural questions in the interview tab out loud. Record yourself once.
  • Identify weakest topic from weeks 1–6. Spend 2 full days on it.
  • Revise all your "stories" — HCSC payment service, ClearCarbon ownership, Citi lead role, Whirlpool Kafka base project. Each should be a crisp 2–3 min STAR format answer.
  • Practice explaining your Spot Award situation, Citi VP interactions, and PIP management — good leadership signals.
WEEK 8 Final Polish + Interview Week Execution ▼
  • Light revision only — no new topics. Review your notes from all 7 weeks.
  • Solve 1 LeetCode easy per day (confidence, not challenge).
  • Re-read your resume — be ready to talk about every line for 5 minutes.
  • Prepare 5 questions to ask the interviewer (about tech stack, team structure, deployment cadence, on-call, etc.)
  • Rest well 2 days before interview. Don't cram.
